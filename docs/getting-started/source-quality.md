# Source Quality Rules

## Source priority

1. Official Cisco/Webex documentation
2. Webex Developer documentation
3. Cisco release notes and changelogs
4. Cisco community answers from Cisco employees or accepted solutions
5. Internal lab notes or customer-specific notes

## Required metadata

Every article should include a source block:

```yaml
product_area: Webex Calling
last_reviewed: 2026-06-04
source_type: Official documentation
sources:
  - https://help.webex.com/
  - https://developer.webex.com/
```

## Review cadence

- Critical operational guides: monthly
- API articles: monthly or when changelog updates
- Admin how-to articles: quarterly
- Glossary/reference pages: semiannual
