# Escalation Checklist

Before escalating, confirm the following items are complete.

## Required

- [ ] Product area is identified.
- [ ] Scope is clearly defined.
- [ ] Reproduction steps are documented.
- [ ] Expected behavior is documented.
- [ ] Actual behavior is documented.
- [ ] Date, time, and timezone are included.
- [ ] At least two examples are collected when possible.
- [ ] Screenshots or error messages are attached.
- [ ] Recent changes are listed.
- [ ] Official documentation has been checked.

## Product-specific evidence

| Product area | Evidence |
|---|---|
| Calling | Caller, called number, direction, location, device/app, call examples |
| Contact Center | Agent, queue, entry point, flow, ANI, DNIS, interaction examples |
| Devices | Model, serial/MAC, firmware, workspace/user, network details |
| APIs | Endpoint, method, status code, response body, auth type, scopes |
| Meetings | Meeting number/link, host, participant, join method, error screenshot |

## Escalation summary format

```text
Customer/Org:
Product Area:
Impact:
Affected Users/Locations:
Start Date/Time/Timezone:
Expected Behavior:
Actual Behavior:
Examples:
Troubleshooting Completed:

Attachments/Evidence:
```
