# Troubleshooting Home

Use this area when Webex is not behaving as expected and you need a repeatable path to isolate the issue.

!!! tip
    Start with scope: who is affected, what service is affected, when it started, and what changed recently.

## Standard troubleshooting flow

1. Confirm the exact product area.
2. Determine scope and impact.
3. Check current service status and recent changes.
4. Compare a working user/device/location with a non-working one.
5. Collect logs, timestamps, screenshots, and examples.
6. Validate against official Cisco/Webex documentation.
7. Escalate only after the issue is reproducible or clearly documented.

## Required evidence for most tickets

- Customer or organization name
- Affected user, workspace, device, queue, flow, or API integration
- Exact date, time, and timezone
- Expected behavior
- Actual behavior
- Screenshots or error messages
- Recent changes
- Troubleshooting already performed
