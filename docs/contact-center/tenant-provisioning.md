# Tenant Provisioning

**Product area:** Webex Contact Center  
**Audience:** Partner administrators, implementation engineers  
**Last reviewed:** 2026-06-04  
**Primary source:** Webex Help Center — Get started with Webex Contact Center: <https://help.webex.com/en-us/article/nee1mb6/Get-Started-with-Webex-Contact-Center>

## Overview

Tenant provisioning is the initial setup process for a Webex Contact Center customer. Cisco documentation describes accessing the Services Setup wizard through Control Hub at `https://admin.webex.com/` and completing setup steps such as confirming license information and setting up location details.

## Pre-provisioning checklist

Before setup, confirm:

- Customer organization exists in Control Hub.
- Contact Center order/licensing is visible.
- Partner/customer administrator access is ready.
- Customer region and location details are known.
- Telephony/PSTN design is known.
- Initial admin users are identified.
- Go-live timeline is understood.

## Setup validation checklist

After provisioning:

- Confirm Contact Center service is visible in Control Hub.
- Confirm admin users can access Contact Center administration areas.
- Confirm license counts.
- Confirm initial users can be assigned Contact Center access.
- Confirm entry point/queue/flow configuration can be created.
- Confirm test call path, when telephony is ready.

## Common mistakes

| Mistake | Impact |
| --- | --- |
| Starting setup without confirmed telephony design | Rework later in the project. |
| Not validating admin access | Delays configuration work. |
| Assuming old Management Portal instructions are current | Inaccurate steps due to migration into Control Hub. |
