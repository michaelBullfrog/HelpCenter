What Control Hub is used for

Control Hub is the central admin portal for managing Webex users, permissions, services, analytics, calling, messaging, meetings, and devices.

  • Manage users, groups, and licenses.
  • Assign administrator roles and permissions.
  • Configure locations, workspaces, calling, meetings, and devices.
  • Review analytics, service health, and troubleshooting data.

Admin role checklist

Before giving a user admin access, confirm what they actually need to manage.

  • Full administrator: Broad organization-level access.
  • Read-only administrator: Good for auditing and support visibility.
  • Support administrator: Useful for troubleshooting without full configuration rights.
  • Location administrator: Best when a user only manages specific locations.
  • Device administrator: Focused on device and workspace management.
  • Compliance officer: Used for eDiscovery/compliance-focused access where applicable.

User and licensing checklist

  1. Confirm the user exists in Control Hub.
  2. Confirm the correct license template or manual license assignment.
  3. Confirm the user has the right location.
  4. Confirm calling, meeting, messaging, or Contact Center licenses as needed.
  5. Confirm email activation and identity provider status if SSO is enabled.

Calling setup overview

Webex Calling setup normally starts with locations, PSTN decisions, phone numbers, users, devices, and calling features.

  1. Create or confirm the location in Control Hub.
  2. Configure PSTN connectivity and emergency address information.
  3. Add, port, or assign phone numbers.
  4. Assign calling licenses to users.
  5. Assign phone numbers and extensions.
  6. Add devices or use soft clients.
  7. Configure call queues, auto attendants, hunt groups, schedules, and voicemail as needed.

Common calling features

  • Auto attendant: Menu-based routing for main numbers.
  • Call queue: Distributes calls to agents or staff members.
  • Hunt group: Rings a defined list of users in a selected pattern.
  • Voicemail group: Shared voicemail target for teams.
  • Business hours schedule: Controls open, closed, and holiday routing.

Call quality intake checklist

  • Caller and called number.
  • Date, time, and timezone of the issue.
  • One-way audio, no audio, dropped call, delay, jitter, or robotic audio.
  • Device type: desk phone, Webex App, MPP phone, mobile, headset, or network device.
  • Network path: wired, Wi-Fi, VPN, home internet, or office internet.
  • Whether the issue affects one user, multiple users, one site, or all sites.

Contact Center setup overview

Webex Contact Center setup generally includes tenant provisioning, users, teams, multimedia profiles, queues, entry points, flows, routing strategies, desktop profiles, and reporting access.

  1. Confirm tenant and licensing.
  2. Confirm Control Hub roles and Contact Center privileges.
  3. Create users, teams, and profiles.
  4. Create queues and entry points.
  5. Build and test flows.
  6. Validate Agent Desktop and supervisor access.
  7. Validate reporting, recording, and retention requirements.

Flow build checklist

  • Identify entry point and DNIS.
  • Confirm business hours and holidays.
  • Define IVR prompts and menu options.
  • Map each menu option to a queue, transfer, callback, or message.
  • Define error handling and timeout routing.
  • Test open, closed, holiday, timeout, and failure scenarios.

Agent Desktop support checklist

  • Confirm the user has the correct Contact Center license.
  • Confirm the user is assigned to the correct team.
  • Confirm desktop profile, multimedia profile, and agent profile.
  • Confirm the dial number or extension is valid.
  • Check browser compatibility, pop-up permissions, and audio permissions.
  • Collect screenshots of login, state, and error messages.

Device assignment basics

Webex Calling devices can be assigned to users or workspaces in Control Hub. Devices can commonly be added by MAC address or activated using an activation code.

  • User device: Personal device assigned to one user.
  • Workspace device: Shared device assigned to a room, lobby, warehouse, or common area.
  • Activation code: Useful when staging a phone without manually entering all details.

Phone registration troubleshooting

  1. Confirm the device is supported and assigned correctly.
  2. Confirm MAC address or activation code.
  3. Confirm the phone has IP address, gateway, DNS, and internet access.
  4. Reboot the phone after assignment changes.
  5. Check whether the issue follows the user, device, network port, or location.

API authentication basics

Webex API requests require an access token in the Authorization header. Personal access tokens are useful for testing, while production applications should use OAuth integrations, bots, or other approved app patterns depending on the use case. 

Authorization: Bearer YOUR_ACCESS_TOKEN

Common API errors

  • 401 Unauthorized: Token is missing, expired, malformed, or invalid.
  • 403 Forbidden: Token works, but the user/app lacks scope, role, license, or permission.
  • 404 Not Found: Resource ID, org ID, person ID, or endpoint may be wrong.
  • 429 Too Many Requests: The app is being rate limited. Add backoff and retry logic.

OAuth troubleshooting checklist

  • Confirm the redirect URI exactly matches the developer portal configuration.
  • Confirm the app is requesting the correct scopes.
  • Confirm the authorization code is only used once.
  • Confirm the client secret has not expired or been rotated.
  • Log token response errors without exposing secrets publicly.

Meetings

Meeting issue intake checklist

  • Meeting link or meeting number.
  • Host and affected participant.
  • Browser or Webex App version.
  • Audio path: computer audio, call me, call in, device, or headset.
  • Video/camera model and permission status.
  • Error message and screenshot.

Calendar and join button checks

  • Confirm calendar integration is enabled.
  • Confirm the meeting invite contains a valid Webex meeting link.
  • Confirm room/device calendar is accepting the invite.
  • Confirm timezone and meeting start time.

Troubleshooting

Calling: no audio or one-way audio

  1. Confirm whether the issue is inbound, outbound, or both.
  2. Test from a different device and network.
  3. Check headset, microphone, and speaker permissions.
  4. Capture examples with caller, called number, time, and timezone.
  5. Check Control Hub troubleshooting data where available.

Calling: wrong routing path

  1. Confirm the number being dialed.
  2. Identify whether the number routes to a user, workspace, auto attendant, call queue, or hunt group.
  3. Check business hours, holiday schedules, and after-hours routing.
  4. Check forwarding rules and overflow settings.
  5. Place a controlled test call and document each hop.

Contact Center: agent not receiving calls

  1. Confirm agent is logged in and in an available state.
  2. Confirm the agent is assigned to the correct team and queue path.
  3. Confirm the flow routes to the expected queue.
  4. Confirm capacity, skill, profile, and multimedia settings.
  5. Test the entry point and document the call path.

Contact Center: flow not behaving as expected

  1. Confirm DNIS and entry point mapping.
  2. Confirm the published flow is the intended version.
  3. Test each IVR option, timeout, invalid entry, and closed-hours branch.
  4. Validate queue names, transfer destinations, and variables.
  5. Collect a timestamped call example for escalation.

Devices: phone will not register

  1. Confirm phone model and MAC address.
  2. Confirm user or workspace assignment.
  3. Verify network connectivity, DNS, and internet access.
  4. Reboot the device after assignment changes.
  5. Factory reset only after validating assignment and network basics.

APIs: 401 or 403 errors

  1. Confirm the Authorization header uses Bearer.
  2. Confirm the token is not expired.
  3. Confirm the token includes the required scopes.
  4. Confirm the authenticated user has the required admin role and license.
  5. Confirm the org ID or resource ID belongs to the accessible org.

Meetings: user cannot join

  1. Confirm the meeting link is valid and current.
  2. Try Webex App and browser join.
  3. Check browser permissions for camera and microphone.
  4. Confirm the user is not blocked by network, proxy, or endpoint security restrictions.

Escalation packet template

  • Customer name and Webex org ID if available.
  • Affected users, devices, queues, numbers, or meetings.
  • Exact date/time/timezone of issue examples.
  • Expected behavior vs actual behavior.
  • Recent changes made in Control Hub, network, carrier, or Contact Center config.
  • Screenshots, call examples, HAR logs, console logs, or device logs where applicable.
  • Business impact and urgency.
